Теория (вопросы)

1. Что такое ООП?
Объектно-ориентированное программирование - это метод программирования, 
основная идея которого заключается в представлении программы в виде совокупности объектов.
Объекты могут взаимодействовать между собой. 
Каждый объект в данном подходе является экземпляром конкретного класса. 
Классы могут образовывать иерархию наследования.

2. Какие принципы ООП вы знаете?
Абстракция, инкапсуляция, наследование, полиморфизм.

3. Что такое абстракция?
Абстракция - это когда мы уделяем внимание только основным для задачи деталях 
и опускаем всё лишнее. 
В ООП абстракция означает, что для каждого объекта мы устанавливаем минимальное количество методов и полей, 
которые будут задействованы в задаче. 
4. Что такое инкапсуляция?
Инкапсуляция - один из принципов ООП, который дает возможность предоставить пользователю методы, 
которые нужны для работы с объектами, и скрыть от него детали их реализации.

5. Какими ключевыми словами неявно помечаются все методы в интерфейсе?
abstract public

6. Какие модификаторы доступа вы знаете? Какой уровень защиты предоставляет каждый из них?
В Java существуют 4 модификатора доступа:
- public - Класс, поле или метод, помеченные этим модификатором доступа, будут доступны классам из любого пакета в программе.
- protected - Поля и методы с таким модификатором доступа будут видны всем классам в текущем пакете и классам, которые наследуют данный класс.
- default - По-другому его называют модификатор по умолчанию. 
Поля и методы с таким модификатором доступа будут видны всем классам в текущем пакете.
- private - Полная противоположность модификатору доступа public. 
Поля и методы с таким модификатором доступны только из кода внутри этого же класса.

7. Какое ключевое слово нужно написать в классе, чтобы использовать интерфейс?
implements 

8. Как должны помечаться все методы из интерфейса, когда мы используем их в классе?
@Override

9. Сколько интерфейсов может применить один класс?
Один класс может применить множество интерфейсов. 

10. Можем ли мы создать экземпляр интерфейса?
Нет
